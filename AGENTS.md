이 프로젝트는 현재 GitHub Pages로 배포 중이며, 실제 사용 중인 앱입니다.
현재 앱의 구조와 기능은 절대 변경하면 안 됩니다.

작업 목표:
- 유지보수와 향후 업그레이드 안정성을 위해 파일만 분리하여 정리한다.
- 분리 안정화가 끝난 뒤에만 후속 수정 작업을 한다.
- 지금 단계에서는 기능 추가, 개선, 최적화, 리팩터링 판단 변경을 하지 않는다.

절대 금지:
1. 임의 변경 금지
2. 추정 변경 금지
3. 동작 변경 금지
4. UI/디자인 변경 금지
5. 함수명, 변수명, id, class, data-속성, DOM 구조 임의 변경 금지
6. 이벤트 연결 방식 임의 변경 금지
7. localStorage key 변경 금지
8. GitHub Pages 배포 경로 깨질 수 있는 상대경로/절대경로 임의 변경 금지
9. CDN 제거/교체 금지
10. PWA 관련 작업 선행 금지
11. 파일 분리와 직접 관련 없는 코드 정리, 포맷팅, 성능개선 금지

작업 원칙:
- 현재 동작을 100% 유지한 상태에서 파일만 분리한다.
- 한 번에 크게 바꾸지 말고, 단계별로 아주 작게 나눈다.
- 매 단계마다 변경 전/후 영향 범위를 설명한다.
- 불확실한 부분은 절대 추정해서 바꾸지 말고, "확인 필요"로 남긴다.
- 변경이 필요한 경우에도 먼저 변경 계획과 영향 범위를 제시하고, 그 다음에만 수정한다.
- 가능한 한 기존 코드 문장과 순서를 보존한다.
- 새 파일로 옮길 때도 코드 내용은 최소한만 건드린다.
- 각 단계 종료 시 반드시 아래를 보고한다:
  1) 생성된 파일 목록
  2) 이동된 코드 범위
  3) 변경된 참조(script/link) 목록
  4) 기능 변경 여부: "없음" 또는 구체적 설명
  5) 확인이 필요한 항목

출력 방식:
- 어려운 설명 말고 쉬운 말로 설명한다.
- 작업 전에는 무엇을 할지 짧게 말한다.
- 작업 후에는 무엇을 옮겼는지 짧게 정리한다.
- 임의 판단은 하지 않는다.
